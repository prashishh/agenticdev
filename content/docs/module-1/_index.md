Module 1 covers picking a harness, installing it, and running your first agent session on a real codebase.

**What you'll learn:**
- Evaluating and selecting a harness for your team
- Setting up an opinionated starter framework
- Completing a scoped task end-to-end
- Orchestrating multiple agents with tools like [Conductor](https://www.conductor.build/) and [tmux](https://tmuxai.dev/)
- Team debrief on what worked and what didn't

**Frameworks being evaluated:** [vibe](https://github.com/prashishh/vibe), [gstack](https://github.com/garrytan/gstack), and OpenCode, each taking a different philosophy toward organizing agent-assisted development. We'll also demonstrate orchestration tools like [Conductor](https://www.conductor.build/) and similar platforms for managing multi-agent workflows alongside [tmux](https://tmuxai.dev/) for terminal-based agent multiplexing.
